- **FR-9** — `/admin/actors/import` flow (template link → styled picker/drop zone → preview → gated confirm → live-region result) + Actors toolbar entry.

## 8. Historical Notes

- **The spec's biggest catch was outside its own scope:** T-9's live verification surfaced a 500 that turned out to be a **latent bootstrap-era defect** — serverless-http's synthetic request (`complete: true`) makes body-parser 2.x skip parsing, so raw Buffers reached the ValidationPipe and crashed class-validator's whitelist. **Every authenticated JSON write on the deployed Lambda had been silently broken since bootstrap**; no prior test or live check ever exercised an authenticated body through the real handler (supertest e2e bypasses serverless-http; earlier live checks were GETs + 401s). Fixed with a shared body-normalizing middleware + a non-object-body 400 guard in the pipe, and permanently guarded by `src/test/lambda-handler.e2e.spec.ts` — the repo's first test that drives the real Lambda entrypoint.
- Byte-stable template generation required pinning ZIP-entry mtimes via a wrap of exceljs's private ZipWriter (JSZip stamps `new Date()` on auto-created folder entries); reviewer verified the guard fails loudly on an exceljs bump.
- Live user review also drove adjacent admin-shell fixes committed outside this spec: sidebar Import/Export placeholder removal, mobile hamburger menu + stacked body (the always-row flex had been crushing `<main>` on phones), brand link to `/admin/actors` (no `/admin` index exists), and a mobile-reachable "View public site" link.
- Package-level `CLAUDE.md`/`AGENTS.md` children for `backend/` and `frontend/` were created at archive time, capturing this spec's hard-won conventions (shared-bootstrap discipline, handler-harness rule, type-fidelity, token discipline, generated-asset rules).
